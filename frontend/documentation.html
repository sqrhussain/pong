Topic

Multiplayer Pong Game

Description

We implemented the early arcade game “pong” with web technologies, to play in the browser. Instead of local multiplayer like the original game, we are to be capable of online multiplayer from one web browser versus another web browser, connected via a server. It is also possible to look up your current highscore after playing the game, connected with your username.

Application Technologies

The main technology in use is be HTML5 with javascript on the client side, following the Observer Pattern. In order to serve this frontend code we need a very basic web server. To display our game graphics we used canvas. As for the storage technology we used Local Storage to save the username. Multiplayer communication is be done over WebSockets, for this we implemented a server in python 3 that takes care of relaying information between browsers. We used two HTML5 features on our website: Audio(several times over the website) and Web forms with placeholder (on the start page for the playername).

How to run the game

In the frontend folder open the index.html in the browser or use python -m SimpleHTTPServer 8080 and open http://localhost:8080 in your browser. In the server folder run pip3 install -r server/requirements.txt to install requirements and run python3 pong.py to run the server.
